## Product Bible 사용 원칙

이 문서는 SAJU WAR의 실제 서비스를 구현하기 위한 기준 문서다. Brand Bible이 "왜 존재하는가"를 정의했다면, Product Bible은 "사용자가 어떤 감정으로 어떤 화면을 지나 어떤 행동을 하게 되는가"를 정의한다.

모든 개발자는 화면을 만들기 전에 이 질문에 답해야 한다.

1. 이 화면은 사용자의 어떤 불안을 어떤 명료함으로 바꾸는가?
2. 사용자는 이 화면을 나갈 때 자기 삶을 더 잘 다룰 수 있다고 느끼는가?
3. AI는 점쟁이처럼 말하는가, 전략가처럼 돕는가?
4. UI는 사주 사이트처럼 보이는가, 개인 작전실처럼 느껴지는가?
5. 기능은 결제로 몰아가는가, 성장과 전략으로 이끄는가?

SAJU WAR의 모든 화면은 "사주를 보는 곳"이 아니라 "내 인생 전략을 배우는 곳"이어야 한다.

# Home

## PAGE NAME

Home

==================================================

## 1. 이 페이지의 존재 이유

Home은 SAJU WAR의 첫 작전실 문이다. 사용자가 처음 들어온 순간, 기존 사주 사이트에서 느끼던 가벼움, 광고성, 불안, 복잡함을 완전히 끊어내야 한다. 이 페이지의 존재 이유는 브랜드를 설명하는 것이 아니라 사용자의 인식 프레임을 바꾸는 것이다.

사용자는 "사주 보러 왔다"라는 마음으로 들어올 수 있다. Home은 그 마음을 "내 무기를 찾으러 왔다"로 변환해야 한다. 그래서 Home은 랜딩 페이지이면서 첫 번째 경험 디자인이다. 이 화면에서 실패하면 뒤의 AI Report, Quest, Academy, Consultation은 모두 사주 부가 기능처럼 느껴진다.

플랫폼에서 Home의 역할은 네 가지다.

- 카테고리 전환: 사주 사이트에서 AI Life Strategy Platform으로 인식 전환
- 첫 행동 유도: 무료 Weapon 분석 시작
- 신뢰 형성: 공포 마케팅이 아닌 품격 있는 자기 이해 서비스로 포지셔닝
- 세계관 진입: 사용자가 주인공이고 AI가 전략가라는 구조를 직관적으로 전달

## 2. 사용자 목표

사용자는 Home에서 다음을 얻어야 한다.

- "이 서비스는 기존 사주 사이트와 다르다"는 즉각적 감각
- 내 삶을 더 잘 이해할 수 있을 것 같다는 기대
- 무료로 시작해도 가치가 있을 것 같다는 확신
- 어떤 정보를 입력하면 어떤 결과를 받는지에 대한 명확함
- 내 무기가 무엇인지 알고 싶다는 호기심

Home의 성공은 CTA 클릭률만이 아니다. 성공한 Home은 사용자의 내면에 하나의 질문을 남긴다.

"내 무기는 무엇일까?"

## 3. 사용자 감정

페이지 진입 전:

사용자는 호기심과 의심을 동시에 가진다. "AI 사주? 또 흔한 운세 사이트인가?"라는 방어적 태도가 있다. 사주에 관심은 있지만, 너무 가볍거나 무섭거나 촌스러운 경험을 경계한다.

↓

첫 화면:

검은 공간, 금빛 명식 그리드, 짧은 문장 "Find Your Weapon."을 본다. 감정은 의심에서 정지로 바뀐다. 너무 많은 말보다 강한 정적이 먼저 온다. 사용자는 "어, 이건 내가 알던 사주 사이트가 아닌데"라고 느껴야 한다.

↓

스크롤:

AI Report, Strategy Map, Academy, War Room 미리보기가 영화 예고편처럼 이어진다. 사용자는 단순 결과 조회가 아니라 하나의 생태계를 본다. 감정은 몰입과 신뢰로 이동한다.

↓

버튼 클릭:

"내 무기 찾기" 버튼은 테스트 시작이 아니라 작전 개시처럼 느껴져야 한다. 클릭 순간 금빛 라인이 버튼에서 입력 플로우로 이어지며 의식적인 시작감을 준다.

↓

결과 확인:

Home에서 바로 결과를 보여주지 않더라도 "분석이 시작되었다"는 기대가 있어야 한다. 사용자는 자신이 플랫폼 안으로 들어갔다고 느낀다.

↓

페이지 종료:

사용자는 Home을 떠나는 것이 아니라 Onboarding으로 진입한다. 감정은 "구경 끝"이 아니라 "이제 내 이야기가 시작된다"가 되어야 한다.

## 4. UX Flow

Step 1. 사용자가 URL 또는 광고/콘텐츠 CTA로 Home 진입
Step 2. Hero에서 핵심 메시지와 CTA 확인
Step 3. "내 무기 찾기" 클릭 또는 스크롤
Step 4. 스크롤 시 AI Life Strategy Platform 구조 이해
Step 5. 리포트 미리보기, Strategy Map, Academy, War Room preview 확인
Step 6. CTA 재노출: "무료로 내 무기 분석하기"
Step 7. 클릭 시 Onboarding으로 이동
Step 8. Home 방문 이벤트와 CTA 위치 분석 저장
Step 9. 미완료 이탈 시 리타겟팅/재방문에서 이어가기 CTA 제공

- 모바일: 로고, 시작하기, 메뉴 아이콘

첫 화면 Hero:

- 풀스크린에 가까운 어두운 배경
- 중앙 상단: 작은 카테고리 라벨 "AI LIFE STRATEGY PLATFORM"
- 중앙: H1 "Find Your Weapon."
- 하단 보조 문장: "사주는 미래가 아니라 전략입니다."

- Secondary Button: "세계관 보기"

하단:

- 사용자 여정: 무료 분석 → 리포트 → 퀘스트 → Academy → War Room
- 마지막 CTA: "당신의 작전실을 여세요"
- Footer: 윤리 고지, 개인정보, 회사 정보

## 6. UI Design

Home UI는 Apple의 절제, Netflix의 몰입, Disney의 문턱 경험을 합친다. 화면은 많은 설명으로 설득하지 않는다. 첫 3초는 감각으로 설득하고, 다음 30초는 구조로 설득한다.

컬러:

- Background: Obsidian Black #070707
- Surface: Deep Charcoal #141414 with 72% opacity
- Primary Accent: War Gold #D4AF37
- Text Primary: Bone White #F4F1EA
- Text Secondary: Muted Silver #A6A09A
- Warning/Intensity: Blood Accent #8A1F1F, 제한적 사용

Glass UI:

- Hero 위 카드 프리뷰는 backdrop blur 20px
- border: 1px solid rgba(212,175,55,0.24)
- box-shadow: 금빛 확산보다 얇은 edge light 중심

버튼:

- Primary는 금색 solid가 아니라 black/gold gradient edge를 가진 premium button
- hover 시 내부가 밝아지기보다 테두리 라인이 살아남
- active 시 미세하게 눌리는 0.98 scale

Loading/Skeleton:

- 일반 회색 skeleton 금지
- 어두운 유리 패널 위에 금빛 scan line이 지나감
- 첫 방문 Hero asset 로딩 중에도 브랜드 문장과 CTA는 먼저 보임

## 7. Animation

Hero 진입:

- 0ms: 검은 배경
- 200ms: 아주 약한 금빛 점들이 명식 좌표처럼 나타남
- 600ms: 그리드 라인이 서서히 연결
- 900ms: H1이 위로 8px 떠오르며 fade-in
- 1200ms: 닫힌 Weapon Card가 중앙 뒤쪽에서 천천히 회전

스크롤:

- 섹션 전환은 parallax를 과하게 쓰지 않는다
- 카드들은 viewport 20% 진입 시 y=24에서 0으로 이동
- 배경 입자는 사용자의 시선 집중을 방해하지 않도록 아주 느림

버튼 hover:

- 금빛 테두리가 왼쪽에서 오른쪽으로 한 번 흐름
- 커서는 pointer와 함께 미세한 glow

페이지 전환:

- Home → Onboarding은 화면 전체가 바뀌는 느낌보다 Hero의 닫힌 카드가 앞으로 오며 입력 UI로 변환되는 느낌

## 8. Gamification

Home에서는 게임화를 직접 드러내기보다 암시한다.

- 첫 CTA 클릭: "Operation Started" 이벤트 발생
- Onboarding 완료 전에는 XP를 보여주지 않음
- 첫 분석 후 "First Weapon Discovery +100 XP"를 제공할 준비
- Home preview에서 Level, Quest, Skill Tree 일부만 노출
- 사용자가 "내가 성장하는 서비스구나"를 느끼되 게임 앱처럼 가볍게 보이지 않게 한다

Home의 게임화 메시지:

- "첫 무기를 발견하면 Arsenal이 열립니다."
- "전략을 실행할수록 Skill이 해금됩니다."

## 9. AI 역할

Home에서 AI는 직접 대화하지 않지만, 사용자가 앞으로 만나게 될 AI의 정체성을 예고한다.

AI가 하는 일:

- 방문자 source와 CTA 관심사를 분석해 Onboarding 목적 선택 기본값 추천
- Home preview 문구를 A/B 테스트로 최적화
- 로그인 사용자는 최근 이력에 맞춰 CTA 변경
- 예: "지난번 분석을 이어서 보기", "오늘의 전략 확인하기"

AI가 분석하는 데이터:

- referral source
- scroll depth
- CTA click location
- logged-in user profile
- previous report status
- academy progress

AI가 보여주는 결과:

- 신규: "무료로 내 무기 분석하기"
- 리포트 미완료: "당신의 Arsenal이 아직 닫혀 있습니다"
- 기존 사용자: "오늘의 작전 브리핑이 도착했습니다"

## 13. 예외 처리

Loading:

- Hero 배경 로딩 전에도 텍스트와 CTA 즉시 표시
- preview cards는 skeleton glass panel

Empty:

- content block 누락 시 fallback copy 사용
- 실험 설정 없음: default variant

Offline:

- Home은 정적 콘텐츠로 대부분 표시
- CTA 클릭 시 offline toast: "연결이 회복되면 분석을 시작할 수 있습니다"

Error:

- personalization 실패 시 generic Home
- event tracking 실패는 사용자 경험 차단 금지

Retry:

- CTA 이동 실패 시 버튼 하단에 "다시 시도" 표시

권한 없음:

- Home에는 권한 없음 상태 없음
- 로그인 사용자 데이터 조회 실패 시 비로그인 Home 표시

데이터 없음:

- 기존 사용자이나 report 없음: 신규 분석 CTA

## 14. Accessibility

모바일:

- CTA는 thumb zone에 배치
- Hero 텍스트는 360px 폭에서도 줄바꿈 안정
- background animation은 정보 전달 필수 요소가 아니어야 함

태블릿:

- preview rail은 2열 또는 horizontal snap

PC:

- wide viewport에서 Hero가 과도하게 빈 화면이 되지 않도록 max content width

색약:

- 금색만으로 상태 표시 금지
- 텍스트, 아이콘, border pattern 병행

스크린리더:

- Hero background는 `aria-hidden`
- CTA label은 명확히 "무료 무기 분석 시작"
- 섹션 heading hierarchy 유지

접근성:

- prefers-reduced-motion 지원
- keyboard focus ring은 금색 외곽선과 명도 대비 보장

# Onboarding

==================================================

## PAGE NAME

Onboarding

==================================================

## 1. 이 페이지의 존재 이유

Onboarding은 사용자의 생년월일시를 받는 입력 폼이 아니다. SAJU WAR 세계관에서 "캐릭터 생성"이자 "작전실 입장 절차"다. 기존 사주 사이트가 사용자에게 정보를 요구한다면, SAJU WAR는 사용자가 자기 분석을 시작하는 의식처럼 느끼게 해야 한다.

플랫폼에서 Onboarding은 첫 신뢰를 만드는 구간이다. 사용자는 민감한 생년월일시와 고민을 입력한다. 이때 UI가 조잡하거나 질문이 과하면 사용자는 방어적으로 변한다. 반대로 입력 과정이 품격 있고 명확하면, 사용자는 "이 서비스가 내 데이터를 함부로 다루지 않겠다"는 느낌을 받는다.

## 2. 사용자 목표

사용자는 Onboarding에서 다음을 얻어야 한다.

- 무엇을 입력해야 하는지 쉽게 이해
- 왜 이 정보가 필요한지 납득
- 태어난 시간을 몰라도 진행할 수 있다는 안심
- 자신의 목적에 맞는 분석이 제공될 것이라는 기대

## 1. 이 페이지의 존재 이유

Weapon은 SAJU WAR의 첫 감동 지점이다. 사용자가 생년월일시를 입력한 뒤 처음 받는 결과이며, 이 순간 사용자는 "이 서비스가 나를 어떻게 바라보는지" 판단한다. 기존 사주 사이트가 긴 텍스트로 해석을 쏟아낸다면, Weapon 페이지는 사용자의 핵심 기질을 하나의 상징적 카드로 공개한다.

이 페이지는 제품 전체의 hook이다. 사용자가 여기서 자기 인식을 얻으면 리포트, Strategy Map, Quest, Academy로 자연스럽게 이동한다. 사용자가 여기서 일반론을 느끼면 모든 전환은 약해진다.

## 2. 사용자 목표

사용자는 Weapon에서 다음을 얻는다.

- 나를 설명하는 강력한 첫 문장
- 내 기질을 상징하는 Primary Weapon
- 내 강점과 취약점의 균형 잡힌 이해
- 지금 바로 해볼 수 있는 첫 전략
- 더 깊은 리포트를 보고 싶다는 욕구

## 3. 사용자 감정

페이지 진입 전:

분석 결과를 기다리는 긴장감. "과연 맞을까?"라는 테스트 태도.

↓

첫 화면:

닫힌 카드가 보이고 "당신의 첫 무기가 발견되었습니다"라는 문구가 나타난다. 기대가 올라간다.

↓

스크롤:

사용자는 카드, 핵심 문장, 오행 균형, Blind Spot, 추천 Quest를 차례로 본다. 감정은 놀람에서 자기 반영으로 이동한다.

↓

버튼 클릭:

"카드 열기" 클릭 순간이 첫 피크다. 결과가 갑자기 쏟아지지 않고 한 장면처럼 공개된다.

↓

결과 확인:

사용자는 "맞다/아니다"보다 "이렇게 볼 수도 있구나"를 느껴야 한다. 강점만 칭찬하지 않고 취약점도 품위 있게 말한다.

↓

페이지 종료:

사용자는 더 알고 싶다. 감정은 "확인 끝"이 아니라 "내 Arsenal을 열고 싶다"로 이어진다.

Strategy Map은 SAJU WAR가 운세 앱이 아니라 인생 전략 플랫폼임을 증명하는 핵심 화면이다. 리포트가 "나를 이해하는 문서"라면 Strategy Map은 "내가 어디에 있고 어디로 가야 하는지 보는 지도"다.

## 2. 사용자 목표

사용자는 Strategy Map에서 다음을 얻는다.

버튼 클릭:

↓

결과 확인:

사용자는 다음 7일 행동과 장기 방향을 동시에 본다. 막연함이 줄어든다.

↓

페이지 종료:

사용자는 "이제 무엇을 해야 할지 안다"는 감각으로 Quest 또는 Academy로 이동한다.

## 1. 이 페이지의 존재 이유

Settings는 사용자의 통제감을 보장하는 공간이다. SAJU WAR는 민감한 데이터를 다루기 때문에 설정 화면은 단순 계정 관리가 아니라 신뢰의 핵심이다. 사용자는 언제든 자신의 데이터, 알림, 공개 범위, AI 사용 동의, 결제를 관리할 수 있어야 한다.

## 2. 사용자 목표

- 계정 정보 관리
- birth profile 수정/삭제
- 개인정보와 AI 학습 동의 관리
- 알림 설정
- 구독/결제 관리
- 접근성/표시 설정

# Cross-Product Implementation Rules

## Experience Rules

1. 모든 화면은 사용자가 "사주를 소비한다"가 아니라 "전략을 얻는다"고 느끼게 해야 한다.
2. 공포, 단정, 조급함으로 결제시키지 않는다.
3. 사용자의 취약한 감정은 수익화 대상이 아니라 보호 대상이다.
4. AI는 항상 선택지를 열고, 행동을 제안하고, 사용자의 자율성을 존중한다.
5. 명리 용어는 숨기지 않되, 사용자가 이해할 수 있게 번역한다.

## Design Rules

1. Black/Gold는 브랜드 축이지만 단조로운 한 색조 화면을 만들지 않는다.
2. Glass UI는 가독성을 해치면 사용하지 않는다.
3. Motion은 의미 있는 순간에만 사용한다.
4. 모바일에서 CTA는 엄지 영역에 둔다.
5. 카드 안에 카드를 중첩하지 않는다.
6. Premium은 화려함보다 절제에서 나온다.

## AI Rules

1. AI는 사주 계산을 직접 하지 않는다. 계산은 엔진이 한다.
2. AI는 엔진 결과를 해석하고, 설명하고, 전략화한다.
3. 모든 AI output은 schema validation과 safety review를 통과한다.
4. 의료, 법률, 투자, 죽음, 재난, 질병 단정 금지.
5. AI의 마지막 출력은 가능하면 다음 행동으로 이어진다.

## Engineering Rules

1. TypeScript strict.
2. Zod schema for API and AI output.
3. Supabase RLS required.
4. Prompt versioning required.
5. Events and analytics are part of product, not afterthought.
6. Loading, Empty, Error states are design requirements.
7. Accessibility is not optional.

## Final Product Principle

SAJU WAR는 사용자의 미래를 대신 말해주는 서비스가 아니다. SAJU WAR는 사용자가 자기 자신을 이해하고, 자신의 무기를 발견하고, 오늘 더 좋은 선택을 하도록 돕는 작전실이다.

사용자가 어떤 화면에 들어오든, 마지막에는 이 감각을 가져야 한다.

"나는 정해진 운명에 갇힌 사람이 아니라, 전략을 배울 수 있는 사람이다."

당신에게도 무기가 있습니다.
